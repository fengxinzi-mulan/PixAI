# PixAI 实现总结

## 当前能力

- Electron + React + TypeScript 桌面应用，主进程负责 IPC、设置持久化、SQLite 历史库、图片请求、文件保存、复制与下载。
- 渲染进程提供三栏工作台、独立图库、设置面板、图片预览、白天/黑夜主题。
- 图片生成接口为 OpenAI-compatible `${baseURL}/v1/images/generations`，默认模型保留 `gpt-image-2`。

## 生成行为

- 当生成数量 `n > 1` 时，主进程会并发发送 `n` 个单图请求。
- 每个请求独立维护 `AbortController`，取消某一张只会取消对应 `requestIndex`。
- 每张图片完成后立即写入 SQLite 和本地图片目录；渲染层在生成期间轮询刷新当前会话，因此完成的图片会立即显示。
- `image_history.request_index` 记录图片对应的原始请求位置，工作区按该字段把成功或失败结果放回原卡片位置，不追加、不换位。
- 用户取消的图片不会写入失败历史，也不会在工作区或图库显示失败卡片。
- 生成结果用时最少记录为 `1s`，避免快速返回时展示 `0s`。

## 预览与工作区

- 工作区图片卡片固定为 3:4 缩略图，卡片上的复制、下载、收藏、删除按钮竖向排列。
- 图片预览窗口支持滚轮缩放、重置、复制、下载、关闭。
- 预览窗口按实际图片展示区域计算初始缩放，减少横图和大图的边缘空白。
- 预览支持上一张/下一张切换，按钮位于图片左右两侧，并支持键盘左右方向键。

## 会话与布局

- 左侧会话列表为紧凑单行布局，文本末尾省略，删除按钮固定在最右侧。
- 左侧会话区支持拖拽调整宽度，宽度保存到 `localStorage`。
- 全局滚动条隐藏，但滚动能力保留，包括 prompt 输入框、工作区、图库和预览提示词。

## 数据结构要点

- `settings.json` 保存服务配置和 API Key 存储状态。
- `pixai.sqlite` 保存会话、生成轮次和历史图片。
- `image_history.request_index` 用于把并发生成结果映射回原始卡片位置。
- 开发环境生成图片文件保存到项目目录 `data/images`，打包后保存到安装目录下的 `data/images`。
